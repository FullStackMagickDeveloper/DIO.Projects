História dos computadores

Os primeiros computadores da 1° a 3° Geração no periodo de 1944 a 1970 foram criados especicamente para pata uso militar, em perídodo da guerra fria, onde a força militar que tivesse o melhor equipamento estaria um passo a frente durante a guerra, implementando melhores estratégias de combate com uso estrategico de computatores que pudessem calcular a rotas de misseis.

Apartir da 4° Gereção em 1971foi lançado o (Personal Computer) o famoso (PC) como é conhecido atualmente, graças ao saldoso Bill Gates esta nova geração de computadores puderam fazer parte de nosso cotidiano, no trabalho, estudos e no lar.

Mas isso só foi possível graças a evolução da comunicação que nos troxe a internet, a internet surgiu através dos sistemas de comunicação.
O primeiro sistema de comunicação foi o telégrafo, em 1958 com utilização do código morce, atravé de uma cabo instalado de um continente a outro pelo oceano.


Em 1950, teve a guerra fria entre Estados Unidos e União Soviética, a corrida espacial, primeiro país a chegar na lua, o (Eniac) foi construido e usado para fazer calculos balísticos, calculando rotas de mísseis. Em meio a guerra surgiu a (DARPA) -> [Agencia de Defesa de Projetos e Pesquisas Avançados]).
Na época (LickLiker) pai da (Cloud Computing) teorizou uma rede na qual todos poderiam se comunica através dela, implantando esta idéia.
Depois de alguns anos de evolução baseados na idéia de LickLiker, veio a surgir a (ARPANET) Sistema de comunicação em pacotes, enviados um a um permitindo multiplos destinos.

A (ARPANET) fez a primeira conexão entre 2 pontos em (29/10/1969) entre ([ULCA] -> University of California) e ([SRI] -> Stanford research Institute), onde foi enviado a palavra(Login)mas só chegou (Lo).sendo a primeira conexão e a primeira queda de conexão ao mesmo tempo.
A (ArpaNet) foi construindo diversos pontos de conexão, então (Vint Cerf) e (Robert Kahn) criaram juntos o (TCPIP) que foi implantado na (ArpaNet), criando mais robustez. Através dessa rede foram criadas micro redes, a (ArpaNet) foi usada com fins de pesquisas científicas e militares, baseado nela surgiu a (MilNet) para uso exclusivo Militar.

A (NSFNET) voltada para notícoas, (NASA), (ESNET), com a mesma finalidade, pouco tempo depois em (1989) surgiu o (Tim Bernes-Lee) que viu a oportunidade de unir o (hyperText) com (TCPIP) e criar o primeiro navegador chamado (WorldWideWeb).
Em (06-Agosto-1991) ele publicou a primeia página Web em (http://info.cern.ch)

O que são Clients?

Clients são os consumidores de imformações através da internet, os Clients são representados através de seus computadores interligados usando a internet como meio de comunicação, que fazem requisições via internet para um serviço especifico disponível na rede como um (E-comerce), nele você faz suas compras e solicita o pagamento, ele envia sua requisição para o (servidor-Web) que devolve um cadastro a ser realizado para concluir as compras, ao completar o cadastro e enviar para o (servidor-Web) ele recebe sua requisição e envia seus dado para um [servidor de banco de dados, o (BD)] armazenar seus dados e devolve ao (servidor Web) seu (login) registrado e acessado pelo usuario para o servidor web dar continuidade no processo de pagamento.

O servidor web requisita um servidor de pagamento de cartões para solicitar o pagamento, o servidor de cartões devolve um cadastro de cartão para o servidor web e este repassa para o usuario cadastrar seus dados bancarios de cartões.
O servidor Web requisita as informações para o servidor de cartões para checar se o cartão existe e esta ativo, ao verificar devolve ao usuario as opções de parcelamento e conclusão do pagamento, quando o usuario clicar em Ok e conclui o pagamento, o servidor de cartões confirma o pagamento enviando para o servidor Web os dados de pagamento recibo, nota fiscal, e este devolve ao usuario uma pagina de conclusão do pagameto finalizando o processo de navegação do usuario no servidor web.
O servidor tem o proposito de servir informações, dados nele armazenados para os clientes. A (hospedagem/armazenamento) de um site é feita dentro de um (servidor Web), o servidor retorna arquivos (html, css, javascript, imagens) que são arquivos estáticos para os clients.
O navegador por sua vez gera um arquivo de (cache)armazenando estes arquivos no computador do cliente, para facilitar a renderização execussão da pagina.
(Navegadores/Browsers) são programas criados por empresas utilizados para executar arquivos dos[servidores web (html, javascript, css, imagens, pdf, txt, json)] que seguem padrões (W3C) porem sempre tem uma diferenca de interpretação, também chamados de browsers.

O que são Servers?

Os servers são os servidores, existem diversos tipos de servidores, como os servidores de arquivos, web, email, stream, dados, e firewall.
Os Clients se conectam a internet consumidos informações navegando em sites, através dos (navegadores/browsers) estes fazem as requisições [www.nomeSite.com] ou (.br) aos servidores DNS e estes fazem uma busca interna buscando o IP referente ao nome do site, retornando para o client, o mesmo é feito quando o usuario vai fazer seu login em um determinado site e o servidor web faz a busca desses dados de login em um servidor de banco de dados, retornando as informações para o client.

Como é uma estrutura básica de um HTML?

A estrutura básica de um (html) consiste das seguntes tags:


< !DOCTYPE html>
< html>
< head>
< title>Título da página< /title>
< meta charset="utf-8"/>
< /head>
< body>

(Conteudo da Página)

< /body>
< /html>

[< !DOCTYPE html>]
A tag !DOCTYPE informa ao navegador a versão do HTML que está sendo utilizada no documento. Por exemplo: no HTML5, basta incluir !DOCTYPE html, e assim o navegador já saberá que se trata de um documento na versão HTML5;


[< html lang = "pt-br">... < /html>]
Esta tag é o elemento básico da estrutura do HTML. Assim sendo, ela conterá todos os elementos do seu documento. Todo documento HTML deve iniciar e finalizar com essa tag, O atributo (lang) especifica o idioma do conteúdo do elemento. Exemplos de valores comuns são “en” para inglês, “es” para espanhol, “fr” para francês e assim por diante.


[< head> ... < /head>]
Essa tag delimita o cabeçalho do documento. Não possui nenhum valor visível, porém é capaz de transmitir ao navegador diversas informações muito úteis e essenciais a uma boa apresentação do seu documento HTML.


[< meta charset = "UTF-8">]
Essa tag permite inserir metadados ao seu documento, no caso acima, a informação (charset=”UTF-8″), que garante a compatibilidade do código com os caracteres de padrão latino americano.


[< title> Titulo da Pagina < /title>]
Essa tag define o título da sua página, o nome que aparecerá na sua aba, janela ou guia. Por esta razão, a tag (< title>) é de grande importância para o (SEO);


[< body>...< /body]
Finalmente, a tag que representa o corpo do documento. Em síntese, é nessa tag que todos os elementos visíveis do seu site devem ser inseridos.

Entendendo a diferença entre listas ordenadas e não ordenadas

[Listas não ordenadas]
As listas não numeradas são usadas para listar itens, sem se preocupar com sua sequência. Chamamos de lista de marcadores apenas.

As (tags) usadas para criar uma lista não ordenada são (< ul> e < li>). A tag (< ul>) é usada para definir a lista e a tag (< li>) é usada para cada item da lista.

< ul>
< li>Internet Explorer< /li>
< li>Opera< /li>
< li>Firefox< /li>
< li>Safari< /li>
< /ul>

(Resultado)
  • Internet Explorer
  • Opera
  • Firefox
  • Safari

[Listas ordenadas ou numerada]
As listas ordenadas ou numeradas são usadas para indicar alguma sequência ou numeração.

As tags usadas para criar uma lista não ordenada são (< ol> e < li>). A tag (< ol>) é usada para definir a lista e a tag (< li>) é usada para cada item da lista.

< ol>
< li>São Paulo< /li>
< li>Rio de Janeiro< /li>
< li>Belo Horizonte< /li>
< li>Brasília< /li>
< /ol>

(Resultado)
  1. São Paulo
  2. Rio de Janeiro
  3. Belo Horizonte
  4. Brasília
